## 2024-05-18 - Path Traversal Vulnerability in Electron IPC File Operations
**Vulnerability:** Electron IPC handlers (`fs:readFile`, `fs:writeFile`, `fs:delete`, `fs:mkdir`, `fs:rename`) accept direct file paths (`filePath`, `dirPath`, `itemPath`) from the renderer process without validating if they are within an allowed directory (workspace root).
**Learning:** This is a classic Path Traversal vulnerability in Electron applications where the main process trusts paths provided by the renderer process. An attacker exploiting a cross-site scripting (XSS) vulnerability or a compromised renderer process could read, write, or delete arbitrary files on the user's system by providing paths like `../../../etc/passwd` or `../../../Users/Username/Documents/secret.txt`.
**Prevention:** Always validate that paths requested by the renderer process fall within a securely established and allowed directory (like the workspace root chosen by the user via `dialog.showOpenDialog`). Use `path.resolve` and check if the resulting path starts with the allowed root path. The allowed root path must be securely stored in the main process and not solely controlled by the renderer.
